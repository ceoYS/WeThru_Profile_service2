## 1. 컨셉 방향

### 톤앤무드
위쓰루 메인사이트(다크 프리미엄)와 시각적 일관성을 유지하되, 비즈니스 프로필 상품 특성에 맞게 **"개인 브랜딩의 힘"** 을 전면에 내세운다. 전체적으로 깊은 다크 배경 위에 골드·아이보리 계열 타이포그래피를 얹어 고급스러움을 표현하며, "당신은 이미 검색당하고 있다"는 긴장감을 도입부에서 조성한다.

- **분위기**: Dark Premium × Editorial × Personal Luxury
- **감정 여정**: 위기감(Problem) → 공감(Why Now) → 욕망(Showcase) → 신뢰(Testimonials) → 행동(CTA)
- **금지 요소**: 보라색 그라디언트 on 화이트 배경, AI슬롭 SaaS 템플릿, Inter·Roboto·Arial

### 디자인 영감 적용 방식
- **shadergradient.co**: Hero 배경에 노이즈 메시 그라디언트(또는 CSS 기반 동일 효과) 적용. 정적이지 않고 살아숨쉬는 배경.
- **minimal.gallery**: 타이포그래피 밀도 — 헤드라인을 크게, 서브텍스트를 작게, 여백을 과감하게 사용.
- **godly.website**: 스크롤 인터랙션 — 섹션 전환 시 요소가 슬라이드·페이드 복합 진입.
- **dark.design**: 색감 참조 — 새까만 검정이 아닌 심오한 차콜(#0A0A0A~#111111)에 골드 포인트.

### 섹션 1: Hero

**섹션 목적**
첫 3초 안에 "이 페이지가 무엇을 파는가"와 "왜 나에게 필요한가"를 동시에 전달. 방문자가 스크롤을 계속하게 만드는 첫인상 확보.

**헤드카피**
> 검색당하는 시대,
> 당신의 프로필이
> 계약을 바꿉니다.

**서브카피**
> 명함 한 장을 건넨 그 순간, 상대는 이미 당신의 이름을 검색하고 있습니다.
> WeThru의 비즈니스 프로필은 검색 결과에서도 당신을 대신해 말합니다.

**레이아웃**
- 전체 뷰포트(100vh) 풀스크린
- 배경: shadergradient 스타일 — 딥 차콜(#0C0C0C)에 골드(#B8924A)와 앰버(#D4A853) 계열 메시 그라디언트, 미세하게 움직임
- 왼쪽 50%: 헤드카피(디스플레이 폰트, 대형) + 서브카피 + CTA 버튼 2개 (카카오톡 상담 / 포트폴리오 보기)
- 오른쪽 50%: charlie-yeo.pro 화면을 담은 노트북+모바일 듀얼 목업 이미지. 살짝 기울어진(perspective transform) 3D 효과.
- 하단 가운데: 마우스 스크롤 유도 애니메이션 (부드럽게 위아래 bounce하는 화살표)

**인터랙션 / 애니메이션**
- 배경 그라디언트: CSS `@keyframes` 또는 JS requestAnimationFrame으로 색 위치 서서히 이동 (5~8초 주기)
- 헤드카피: 페이지 로드 시 단어 단위로 0.1s 딜레이를 두고 fade-up 등장
- 목업 이미지: 로드 완료 후 0.4s 뒤 오른쪽에서 translateX(40px) → 0 + opacity 0 → 1
- 마우스 커서가 목업 위에 올라가면 살짝 기울기(rotate) 변화 (CSS perspective hover)
- 배경 그라디언트는 마우스 X/Y 위치에 따라 미세하게 반응 (ParallaxMouse 효과)

### 섹션 2: Problem

**섹션 목적**
방문자의 현재 불안(나는 검색했을 때 어떻게 보이는가?)을 구체적으로 건드려 문제 인식을 강화. "종이 명함"이 가진 한계와 비즈니스 프로필의 차별점을 대비 시각화로 전달.

**헤드카피**
> 명함을 건네고 3초 뒤,
> 상대는 당신을 검색합니다.

**서브카피**
> 그 순간, 당신의 이름이 검색 결과에 아무것도 없다면?
> 혹은 오래된 LinkedIn 사진과 잡다한 SNS만 나온다면?

**레이아웃**
- 다크 배경(#111111) 전환. 배경 노이즈 텍스처 미세하게 깔림.
- 2단 비교 레이아웃 (좌: 종이명함의 현실 / 우: 비즈니스 프로필의 결과)
- 좌측 컬럼: 종이명함 이미지(흑백·저채도 처리) + 아래에 단점 3가지 불릿 리스트
  - "명함함 안에서 잊힌다"
  - "검색해도 아무것도 없다"
  - "첫인상 한 번으로 끝"
- 우측 컬럼: charlie-yeo.pro 목업(컬러·고채도) + 장점 3가지 불릿 리스트
  - "24시간 365일 대신 소개"
  - "QR 하나로 즉시 연결"
  - "업데이트 가능한 살아있는 명함"
- 두 컬럼 사이 중앙에 VS 배지 (골드 원형)
- 섹션 하단: 작은 통계 수치 한 줄 — "첫 미팅 후 84%의 사람이 상대방 이름을 검색합니다"

**인터랙션 / 애니메이션**
- IntersectionObserver: 섹션 진입 시 좌측 컬럼은 왼쪽에서, 우측 컬럼은 오른쪽에서 슬라이드인
- VS 배지: 두 컬럼이 등장한 뒤 scale(0) → scale(1) 팝인
- 불릿 리스트: 순서대로 0.15s 간격으로 fade-up 등장

**색상 팔레트**

|------|------|-----|
| 섹션 배경 | 다크 그레이 | `#111111` |
| 좌(종이명함) 배경 | 더 어두운 그레이 | `#0E0E0E` |
| 우(프로필) 배경 | 딥 차콜 | `#161616` |
| 단점 아이콘 | 둔한 레드 | `#8B3A3A` |
| 장점 아이콘 | 골드 | `#C9A84C` |

### 섹션 3: Why Now

"지금 당장 해야 하는 이유"를 데이터와 트렌드로 설득. 개인 브랜딩은 선택이 아닌 생존 전략임을 공신력 있는 수치로 강화.

**헤드카피**
> 개인 브랜딩은
> 이제 선택이 아닙니다.

**서브카피**
> 시장은 이미 변했습니다. 이름 하나가 곧 신뢰입니다.

**레이아웃**

- 헤드카피 + 서브카피 상단 중앙 정렬
- 그 아래 통계 카드 3개를 가로 배열 (1/3씩 분할, 카드 아닌 오픈 레이아웃):

- 각 카드: 큰 숫자 + 설명 한 줄 + 출처 작은 글씨
- 섹션 하단: 인용구 블록 — 세로 골드 바 + "당신이 구글에서 보이지 않는다면, 당신은 존재하지 않는 것과 같습니다 — Guy Kawasaki"

**인터랙션 / 애니메이션**
- 통계 숫자: IntersectionObserver 진입 시 0에서 최종 숫자로 카운트업 애니메이션 (1.5s)

### 섹션 4: Showcase

**섹션 목적**
실제 납품 사례를 통해 "이런 게 나온다"는 구체적인 기대감을 형성. 스타일이 다양함을 보여줘 "내 스타일로도 가능하다"는 확신 제공.

**헤드카피**
> 세 가지 이야기,
> 세 가지 브랜드.

**서브카피**
> 어떤 직군도, 어떤 스타일도 맞춤 제작됩니다.

**레이아웃**
- 배경: #0F0F0F
- 상단 헤드카피 + 서브카피 좌측 정렬
- 메인 영역: 브라우저 목업 프레임 1개 (전체 너비의 70%) 중앙 배치
  - 목업 프레임 안에 실제 레퍼런스 사이트를 iframe 또는 스크린샷 슬라이드로 표시
  - 프레임 상단에 macOS 스타일 트래픽 라이트 버튼 (장식용)
- 우측에 세로 탭 네비게이션 3개:
  - 01 — Charlie Yeo / Dark Premium
  - 02 — Jehoon Yun / Career Story
  - 03 — Hansol / Minimal Card
- 탭 클릭/호버 시 목업 콘텐츠 전환
- 하단에 각 프로필별 태그 칩 (업종, 톤, 특징): 예) `#다크톤` `#개발자` `#풀페이지스크롤`

**인터랙션 / 애니메이션**
- 탭 전환 시 목업 안 콘텐츠: crossfade (opacity 전환, 0.4s)
- 활성 탭: 왼쪽에 골드 수직 바 표시 + 텍스트 골드 강조
- 목업 프레임 자체: hover 시 미세한 translateY(-4px) + box-shadow 심화
- IntersectionObserver: 섹션 진입 시 목업 프레임 scale(0.95) → scale(1) + opacity

- Problem: 2단 → 상하 스택 (종이명함 위, 프로필 아래). VS 배지 중앙 floating.
- Why Now: 통계 카드 3개 → 세로 스택
- Showcase: 탭 → 드롭다운 선택 또는 스와이프 가능한 수평 스크롤
- Pricing: 3열 → 세로 스택. PRO 카드 첫 번째 표시.
- Process: 세로 단일 타임라인 (교대 없음)
- Testimonials: 카드 1개씩 전체 너비 슬라이드
- Cross-sell: 4열 → 세로 스택 2개씩
- CTA 버튼: 전체 너비(100%)

## 4. 수용 기준 (Acceptance Criteria)

모든 기준은 **7점 이하면 불합격**. 불합격 시 구체적 개선안 제시 필수.

---

### 기준 1: Design Quality (시각적 완성도)
**채점 범위**: 색상·타이포그래피·여백·비율의 조화, 다크 프리미엄 톤 유지 여부

| 점수 | 기준 |
|------|------|
| 10 | 색상 팔레트 완벽 준수, 타이포 hierarchy 명확, 여백 일관성 완벽, 상업 수준 완성도 |
| 9 | 소수 섹션 여백 불균일, 전체적 완성도 높음 |
| 8 | 색상 이탈 1~2곳, 타이포 사이즈 일관성 약간 저하 |
| 7 | **불합격선** — 다크 프리미엄 톤 훼손, 여백 들쭉날쭉, 골드 포인트 남용 |
| ≤6 | 보라 그라디언트 사용, Arial/Roboto 폰트 사용, SaaS 템플릿 느낌 |

**7점 이하 개선안**: 색상 팔레트 변수 재점검 → 허용되지 않은 폰트 전수 제거 → 섹션별 padding/margin 표준값 CSS 변수화 → 실제 레퍼런스 사이트(charlie-yeo.pro)와 나란히 비교하여 톤 교정

---

### 기준 2: Originality (독창성)
**채점 범위**: AI슬롭·제네릭 SaaS 느낌 탈피 여부, 한국 개인 브랜딩 시장에 특화된 디자인 언어 구사 여부

| 점수 | 기준 |
|------|------|
| 10 | minimal.gallery / dark.design 수준의 독창적 레이아웃, 어느 SaaS 사이트와도 다른 느낌 |
| 9 | 2개 이상의 섹션에서 차별화된 레이아웃 아이디어 구현 |
| 8 | 전체적으로 독창적이나 1~2개 섹션에서 뻔한 그리드 사용 |
| 7 | **불합격선** — 히어로 CTA 버튼 2개 나란히, 3열 Feature 카드, 중앙 타이틀 반복 등 SaaS 클리셰 과반 |
| ≤6 | "AI가 만든 느낌" 명확, 대부분의 섹션이 제네릭 템플릿과 구분 불가 |

**7점 이하 개선안**: 각 섹션의 레이아웃 재설계 — 대칭형 카드 그리드 → 비대칭 타이포그래피 레이아웃으로 교체 / 히어로 섹션에 shadergradient 배경 미구현 시 즉시 추가 / godly.website 인터랙션 레퍼런스 재참조 후 스크롤 트리거 재설계

---

### 기준 3: Craft (코드 품질)
**채점 범위**: CSS 변수 활용, 반응형 완성도, 성능(Lighthouse 90+), 코드 가독성

| 점수 | 기준 |
|------|------|
| 10 | CSS 커스텀 프로퍼티로 모든 색상·간격·폰트 관리, Lighthouse 95+, FCP ≤ 1.0s |
| 9 | CSS 변수 활용, Lighthouse 90~94, FCP ≤ 1.5s |
| 8 | 대부분 변수화, 일부 하드코딩, Lighthouse 85~89 |
| 7 | **불합격선** — 색상 하드코딩 다수, Lighthouse 80 미만, 또는 375px에서 레이아웃 깨짐 |
| ≤6 | CSS 변수 미사용, 모바일 레이아웃 미완성, 성능 심각 저하 |

**7점 이하 개선안**: `:root` CSS 변수 블록 전면 도입 → 모바일(375px) 실기기 또는 DevTools 전수 확인 → 이미지 WebP 변환 및 lazy loading 적용 → 불필요한 JS 제거, CSS 애니메이션으로 교체 → Lighthouse CI 체크

---

### 기준 4: Functionality (기능 완성도)
**채점 범위**: 모든 섹션 렌더링, 링크 유효성, 스크롤 애니메이션 정상 동작, 크로스브라우저

| 점수 | 기준 |
|------|------|
| 10 | 모든 링크 유효, 모든 애니메이션 정상, Safari/Chrome/Firefox 정상, 카카오톡 버튼 연결 |
| 9 | 1개 브라우저에서 미세한 시각 차이, 기능 모두 동작 |
| 8 | 1~2개 인터랙션 오작동, 레이아웃 정상 |
| 7 | **불합격선** — Showcase 탭 전환 오작동, 또는 카운트업 애니메이션 미작동, 또는 외부 링크 404 |
| ≤6 | 다수 섹션 레이아웃 깨짐, CTA 버튼 링크 불작동, 애니메이션 다수 미구현 |

**7점 이하 개선안**: 모든 `<a>` href 값 전수 점검 → IntersectionObserver 지원 여부 polyfill 확인 → Showcase 탭 JS 로직 디버깅 (이벤트 위임 방식으로 재작성) → 카카오톡 채널 링크 실제 연결 확인
